post /add_acquisition_user
Adds new acquisition user (addAcquisitionUserPost)

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body add_acquisition_user_body (optional)
Body Parameter —
example: { "summary" : "Add acquisition user with all correct information", "value" : { "acquisition_id" : "some_email@email.com", "acquisition_id_type" : 1, "answers" : [ "iPhone XR", "18-25", "TX" ], "campaign_id" : "1" } }

Request headers

Return type

inline_response_200

Example data

Content-Type: application/json
{
  "code" : 0,
  "module" : "module",
  "acquisition_eligibility" : true
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

Successful response inline_response_200

Example data

Content-Type: success
{"code":0,"module":"COMMON","acquisition_eligibility":true}

Example data

Content-Type: success_for_uninvited_or_wrong_answers
{"code":0,"module":"COMMON","acquisition_eligibility":false}

Bad request #/components/responses/400

Example data

{"code":2,"module":"COMMON"}
