post /get_number_info
Get the subscriber information from the MSISDN (getNumberInfoPost)
By providing application_id, fetch corresponding operator-specific information. Used by Portal

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body get_number_info_body (required)
Body Parameter —

Request headers

Return type

inline_response_200

Example data

Content-Type: application/json
{
  "server_time" : 1548283725,
  "number_info" : {
    "activation_date" : 1533131266,
    "current_billed" : 1,
    "contract_dates" : 1533131266,
    "sim_card_information" : {
      "iccid" : "8901260381754011695",
      "imsi" : "967444610810170"
    },
    "deposit_held" : 0,
    "account_status" : 2,
    "subscription_type" : 2,
    "unbilled_balance" : 0,
    "rate_plan" : 1
  },
  "code" : 0,
  "module" : "PORTING"
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

Fetched operator-specific information successfully inline_response_200

400

Invalid request. This may be due to invalid JSON, or invalid/missing fields in the request body. failed_request_body

403

Authorization failure

500

Server error 500_response
