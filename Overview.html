Overview

This document describes functions, syntax, parameters, and examples of the application programming interfaces (APIs) of LotusFlare DNO Platform.   DNO supports Representational State Transfer (REST) APIs, allowing you to call APIs using HTTPS


URL

LotusFlare APIs are RESTful APIs.

Representational State Transfer (REST) allocates Uniform Resource Identifiers (URIs) to dispersed resources so that the resources can be located. Applications on clients use Unified Resource Locators (URLs) to obtain resources.

RESTful URLs are in the following format:

https://api.lotusflare.com/api/{version}/{rest-path}



Parameter

Description

version

Version number, in format of "v" plus a positive integer (for example, v1). 

rest-path

Resource path in the service.


For example, the API for obtaining basic information about user information is https://api.lotusflare.com/api/v3/authorization/get_subscriber



Request

Request Methods

According to REST, all operations on resources must comply with HTTPS specifications.

Request methods include GET and POST. 



Method

Description

POST

The POST method requests that the server accept the entity enclosed in the request as a new subordinate of the web resource identified by the URI.

GET

The GET method requests a representation of the specified resource.



POST

[Usage scenarios] This method is used to create resources or perform other operations.

[Status codes] If resources are successfully created, status code 200 ("OK") or 201 ("Created") will be returned.


GET

[Usage scenarios] This method is used to obtain resources.

[Status codes] If resources are successfully obtained, status code 200 ("OK") will be returned.


Status Codes

Description

400
