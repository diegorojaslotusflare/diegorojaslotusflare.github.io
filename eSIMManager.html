EMP_profile_states Up

ESIM_Cloud_profile_states Up

<p>Following states are common profile states defined by GSMA. They are the same for all operators, and with them we can conclude what API calls were made on one profile.</p> <ul> <li>&quot;ALLOCATED&quot; - Profile is reserved for downloading without being linked to an EID - download_profile call was made without eid parameter</li> <li>&quot;LINKED&quot; - Profile is reserved for downloading and is linked to an EID - download_profile call was made with eid parameter</li> <li>&quot;CONFIRMED&quot; - Profile is reserved for downloading (linked or not linked to an EID) with Matching ID and Confirmation Code if required - confirm_profile call was made with release_flag=false</li> <li>&quot;RELEASED&quot; - Profile is ready for download and installation after Network Configuration by the Operator (e.g.: HLR Registration) - call cofnrim_profile was called with release_flag=true or release_profile was called</li> <li>&quot;DOWNLOADED&quot; - Bound Profile was delivered to the LPA - notification for successful download was sent to server side</li> <li>&quot;INSTALLED&quot; - Profile was successfully installed on the eUICC - notification for successful installation was sent to server side</li> </ul>

ESIM_Cloud_profile_status Up

<p>Statuses of the internal esim profile that exists in esim cloud database. For one resource_id (identifier), multiple profiles can exist in esim cloud, but only one will be VALID at any moment.</p> <ul> <li>1 - PENDING - profile is still not active, as result of <code>request_new_profile</code> call, waiting waiting for callback (if there is any callback needed, if not, this status will be skipped)</li> <li>2 - CANCELED - profile is not valid anymore, it is canceled as result of <code>cancel_order</code> call</li> <li>3 - VALID - profile valid</li> <li>4 - REPLACED - profile is not valid anymore, replaced with new profile as result of <code>request_new_profile</code> call</li> <li>5 - DELETED - profile is not valid anymore, deleted from device</li> </ul>

ESIM_PROVISIONING_RESOURCES_ID Up

<ul> <li>EXTERNAL_OFFER_ID - offer id provided from operator</li> <li>CFS_ID - feature id used for EDA provisioning</li> <li>LF_OFFER_ID - interal LotusFlare offer id</li> <li>LF_ORDER_ID - internal LotusFlare order id</li> </ul>

ESIM_orchestration_status Up

<ul> <li>SUCCESS = 1</li> <li>FAILED = 2</li> <li>EXECUTING = 3</li> <li>STOPPED = 4</li> <li>PENDING = 5</li> <li>PAUSED = 6</li> <li>WAITING_FOR_CALLBACK = 7</li> </ul>

IDENTIFIERS Up

<ul> <li>ICCID - Integrated Circuit Card Identification Number</li> <li>IMEI - International Mobile Equipment Identity</li> <li>DEVICE_ID - Same device_id that was used when activating/request new esim profile</li> <li>EID - eUICC Identifier</li> <li>MSISDN - Mobile Station International Subscriber Directory Number</li> <li>IMSI - International Mobile Subscriber Identity</li> <li>LF_SUBSCRIBER - LotusFlare Subscriber Identity</li> <li>EXTERNAL_SUBSCRIBER - External Subscriber Identity</li> </ul>

SMDP_profile_states Up

<ul> <li>1 - Allocated</li> <li>2 - Available</li> <li>3 - Confirmed</li> <li>4 - Created</li> <li>5 - DeleteSucceeded</li> <li>6 - DownloadInitiated</li> <li>7 - Error_DeleteFailed</li> <li>8 - Error_DownloadFailed</li> <li>9 - Error_InstallFailed</li> <li>10 - ExpiredAfterDownload</li> <li>11 - ExpiredBeforeDownload</li> <li>12 - Installed</li> <li>13 - Installed_Disabled</li> <li>14 - Installed_Enabled</li> <li>15 - Linked</li> <li>16 - MarkedForDeletion</li> <li>17 - MarkedForDeletionWithHistory</li> <li>18 - Prebooked</li> <li>19 - Released</li> <li>20 - Unavailable</li> </ul>
