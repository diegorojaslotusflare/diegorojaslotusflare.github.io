API Overview

APIs provided by LotusFlare DNO Platform are classified by each DNO component. By using this set of APIs, you can use all functions provided by DNO, including displaying, adding, modifying resources. 



Component

Description

Swagger file name

acquisition

Acquisition APIs are used to add new acquisition user, check acquisition eligibility for a given user, update answers of particular acquisition user.

Acquisition.yaml

authorization

APIs available for role management, permission management, Session Management and Token verification

Authorization.yaml

billing

Billing related APIs for request to generate account billing statements.

Billing.yaml

catalog

API for managing Product Catalog entities

and amendments

ProductCatalog.yaml

cdp



charging

Charging Manager Routes

Charging Entities, Routes

Charging.yaml

content

Management API that allow add, update, remove or list content to be managed

ContentManagement.yaml

delivery

This is a service for delivery, it can be use to get supported delivery methods for delivery address, create delivery, get details about delivery and gets all deliveries for a user

Delivery.yaml

loyalty

API for managing Reward Programs and Registration

LoyaltyProgramManagement.yaml

number

Number Management APIs help to do batch operation of msisdns that are added to number management inventory, remove a msisdn from number management inventory,  request a set of msisdns from number management inventory, reserve a msisdn from number management inventory, takes a msisdn and reserves it, releases a previous reservation done on number management inventory, downloads various numbers with given status from number management inventory.

NumberManagement.yaml

orchestration

Manage different entities within orchestration engine, such as plans, executions, histories, etc., Scheduling and execution manipulation

APIs that are to be used for scheduling of plans, running, stopping and controlling executions in other ways.

Orchestration.yaml

order

Order Management APIs allow the order creation, provisioning of specific entity, activates a previously provisioned pending order, deactivates a previously provisioned active order, and cancel a previously provisioned order

Order.yaml

payment

Payment related APIs help to retrieve client tokens from the third party payment gateway, fetching all payment methods for the customer, and managing payment methods.

Payment.yaml

porting

Port in and out service and help to get the subscriber information from the MSISDN

Porting.yaml

prize

API for Prize Engine.  These APIs are called when a prize engine page is opened in the client app. Based on the app’s context (and the subscriber’s segments), the appropriate prize engine definition is returned to the app. Depending from request auth token prize manager returns public or private representation of prize engine (e.g with segment_expression or without it).

PrizeEngine.yaml

promotion

Promotion Management related APIs.  Used to add a promo rule entity, get one or more promo rules, update a promo rule, or update state of Promo Rule,

PromotionManagement.yaml

referral

Referral Service. It describes APIs for referral service. “Referrer” means existing operator client who participates by referring other people to become Referees. “Referee” means a person who is not an existing client of particular operator and whom the Referrer has referred through the referral code.

Referral.yaml

reward

Reward Module.  APIs are used to returns a dictionary of reward rules, return a list of rewards and prize payouts, sorted in ascending order of time

Reward.yaml

sim

SIM APIs are used to add one iccid-imsi map to sim_data pool, and

validate provided iccid by attempting to lookup imsi

SIM.yaml

tiers

This is the Client API for the Tiers module.  These APIs returns information on all tiers defined in the system, and returns information on the current tier subscriber holds

TiersClient.yaml

top_up

Top Up APIs are used to top up a specific amount for a subscriber, top up from voucher, deduct a specific amount for a subscriber and cancel a top up.

TopUp.yaml

user

User Management provides APIs available for fetching information on user/account/subscriber. Registration

APIs available for registering or login a user in LF system, Address Management

APIs available for address management and manipulation, Roles Management

APIs available for roles management and manipulation, Session Management

APIs available for session management and manipulation, OTP

APIs used when OTP code is sent to the client, General User Manager

APIs available for changing/reading information from the LF system.

User.yaml

vouchers

API for managing Vouchers sets and Voucher set codes.  APIs are used to get a list of all voucher sets and adds additional info, add voucher codes to a voucher set, get a list of subscriber's assigned voucher sets with usage information, gets a list of active voucher sets that reached minimum threshold

Voucher.yaml

waitlist

Waitlist APIs are used to add a new waitlist user,

get waitlist users,

unsubscribe a waitlist user, add a new eligibility address check, get a eligibility check address,

update the eligibility check with acquisition lead

Waitlist.yaml

cpq

CPQ APIs are used to request to search product offering by context, request to search PSR document by spec id, request to search all PSR documents.

cpq.yaml
